# Data Instructions

This repository tracks small metadata and result-support files, but not the full raw smart meter block dataset.

## Expected Layout

```text
data/
  carbon/
    neso_historic_generation_mix.csv
  metadata/
    informations_households.csv
    weather_hourly_darksky.csv
    weather_daily_darksky.csv
    uk_bank_holidays.csv
    acorn_details.csv
    darksky_parameters_documentation.html
  processed/
    features_sample500.parquet          # generated by notebook 01, not tracked
  raw/
    smart_meter_data/                   # downloaded source data, not tracked
      halfhourly_dataset/
        halfhourly_dataset/
          block_0.csv
          ...
          block_111.csv
      daily_dataset/
      hhblock_dataset/
      daily_dataset.csv
```

## Download Sources

- London smart meter dataset: https://data.london.gov.uk/dataset/00326043-62b6-47a8-9a49-1aa226b7e2c4/
- NESO historic generation mix and carbon intensity: https://www.neso.energy/data-portal/historic-generation-mix

The main training notebook can also download the smart meter dataset with `kagglehub` when credentials are available.

## Large File Policy

The full London smart meter dataset is roughly 10 GB unzipped. GitHub blocks regular Git files larger than 100 MiB and large repositories become slow to clone and review. For that reason:

- Code, documentation, small metadata, model metadata, final figures, and final result tables are tracked.
- Large raw data is placed under `data/raw/` and ignored by Git.
- Large derived feature caches are placed under `data/processed/` and ignored by Git.
- Larger model/carbon artifacts are stored through Git LFS where needed.

To reproduce the full pipeline, download the raw dataset into `data/raw/smart_meter_data/` or let `notebooks/01_model_training.ipynb` download it, then run the notebooks in order from the repository root.
